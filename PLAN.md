# Overall Implementation Plan — Task App

## Overview

Real-time multi-user task management app with organizations (groups for family/teams), subtasks, recurring tasks, time tracking, milestones, comments with @mentions, and real-time websocket updates. Backend and frontend developed continuously, interleaved per feature.

## Tech Stack

| Layer | Tech |
|---|---|
| Frontend | Next.js 16.2.9, React 19.2.4, Tailwind v4, Shadcn UI, Zustand, @dnd-kit, react-big-calendar |
| Backend | Fastify 5, Drizzle ORM 0.45, postgres.js, better-auth (org plugin + Google OAuth), ws |
| Shared | `@meu-projeto/types` (Zod schemas + inferred TS types) |
| DB | PostgreSQL (Neon/Supabase) |
| Real-time | ws via @fastify/websocket |

### Step 6: Backend Tasks Core (CRUD + Subtasks)

**Files to create:**

- `src/controllers/tasks.controller.ts` — thin wrapper
- `src/routes/tasks.routes.ts` — zod-validated routes

**Route map:**

```
POST   /api/groups/:groupId/tasks            -> create task (with optional parentId for subtask)
GET    /api/groups/:groupId/tasks            -> list tasks (filters: status, priority, assignee, tag, milestone)
GET    /api/groups/:groupId/tasks/:taskId    -> get task (with subtasks, tags, time entries, comments count)
PATCH  /api/groups/:groupId/tasks/:taskId    -> update task
DELETE /api/groups/:groupId/tasks/:taskId    -> delete task
```

**Verify:** API CRUD works via curl/REST client, subtask creation via `parentId`.

## Execution Rules

1. Each step is executed independently and fully before moving to the next.
2. Backend and frontend are interleaved per feature (not all backend then all frontend).
3. After each step: run typecheck (`tsc --noEmit`), lint (`eslint`), and verify the feature works.
4. Commit after each step is verified working.
5. Never skip verification — each step builds on the previous.
